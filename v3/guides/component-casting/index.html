Component casting

SJSF provides a rich set of interchangeable components, but some of them cannot be used directly out of the box (tagsField, filesField, nativeFileField, etc.). Let’s look at a few ways to work with them.

Ignore errors (not recommended)

Section titled “Ignore errors (not recommended)”

You can suppress typing errors if you just want to quickly test an idea or see how a component looks in place.

Create an adapter component

Section titled “Create an adapter component”

You can create a new component that adapts the interface of the original component (for example, arrayField) to the interface of the replacement component (tagsField).

Why is this needed? Throwing an error when data formats don’t match is only one possible approach.
You should decide on the adaptation strategy and implement it yourself. However, if you’re fine with the assert-based approach, you can use the following prebuilt field components:

  • array-files
  • array-native-files
  • array-tags
  • unknown-native-file

Caution

This utility relies on internal details of Svelte 5 that may change even in a patch release. Use it with care.
